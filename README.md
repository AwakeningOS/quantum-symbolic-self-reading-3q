**自分の物語を、量子象徴回路に通して読む。**

人生相談、創作、信仰体験、キャリアの迷い、恋愛、家族、喪失、転機。  
ふつうに読むと「なんとなくそういう話」で終わる物語を、4つの成分とゲート列に変換し、ブラウザ上の2量子ビット象徴回路で測定します。

## これは何？

**Quantum Symbolic Self-Reading** は、あなたの文章を「物語の構造」として読むための実験的なセルフリーディングツールです。

流れはシンプルです。

```text
あなたの物語
↓
AIが config JSON に変換
↓
ブラウザ内の量子象徴回路で測定
↓
AIが測定結果を物語の言葉に翻訳
```

ポイントは、AIにただ感想を書かせるのではなく、**一度、固定された回路計算に通す**ことです。

AIだけで読むと、回答はどうしても「それっぽい共感」や「一般論」に寄りやすくなります。  
このツールでは、AIが作った物語モデルをブラウザ側で計算し、確率、干渉、順序感度、位相感度、絡み合い、反実仮想を出します。

つまりこれは、

> **自分の物語を読むための、象徴的な量子干渉計**

です。

## 何が面白いの？

このツールが見るのは、単なる「どの要素が強いか」だけではありません。

たとえば、こんなことが見えます。

### 1. 語りと構造のずれ

本人やAIエンコーダは「この物語はこういう結末になるはず」と予想します。  
でも、実際に回路を通すと違う順位になることがあります。

```text
予想: 本当の自分が主役になるはず
実測: まだ家族観・役割規範の方が主成分だった
```

このずれが面白いところです。

「自分がそう語っている物語」と、  
「語られた出来事の列が構造的に導いた物語」が、  
一致しているとは限りません。

---

### 2. 蝶番の出来事

どの出来事を取り除くと、結末が一番変わるのかを調べます。

これは `ablation` と `gate_resonance` で見ます。

たとえば、人生の中では目立たない出来事でも、回路上では「それがなければ全部違っていた」と出ることがあります。

---

### 3. 静かな種

起きた瞬間には小さく見えたのに、あとから大きく効いていた出来事を検出します。

これは `QUIET_SEED` として出ます。

例:

```text
夫の感情表現の乏しさ
子育て中の毎日の疲労
ふと増えた一人時間
昔好きだったことを少し再開した瞬間
```

こういうものは、物語を普通に読んだだけでは見落としやすいです。  
でも回路では、「その瞬間の変化」と「最終結果への重み」を別々に見ます。

---

### 4. 洗い流されたもの

逆に、当時は大事件だったのに、後続の流れによって最終的には薄まった出来事も見えます。

これは `WASHED_OUT` として出ます。

大きな出来事が、必ずしも最後まで物語を支配するとは限りません。

---

### 5. 順序の意味

同じ出来事でも、起きる順番が違えば、まったく別の結末になることがあります。

このツールは隣り合うゲートの順序を入れ替え、最終分布がどれだけ変わるかを測ります。

```text
先に挫折が来たから、才能が目覚めたのか
先に出会いが来たから、恐れを超えられたのか
先に空白が来たから、本音が出てきたのか
```

これが `order_sensitivity` です。

---

### 6. 通り方の質

同じ出来事でも、それをどう通ったかで結果が変わります。

- 受容として通ったのか
- 葛藤として通ったのか
- 拒絶や反転として通ったのか
- 挫折から折り返したのか

これを `phi`、つまり位相として扱います。

`phase_sensitivity` と `classical_controls.phase_dependence` によって、位相が本当に結果を動かしているかを見ます。

---

### 7. 二つの問いの絡み合い

このツールの4成分は、ただの4分類ではありません。  
実は、2つの軸の組み合わせです。

一般ヴァージョンではこうです。

|  | 潜在・背後 | 顕在・現実化 |
|---|---|---|
| 個別存在 | `a` 内的核 | `b` 現実相 |
| 上位構造 | `c` 背後秩序 | `d` 顕在作用 |

つまり、

```text
a = 個別存在 × 潜在
b = 個別存在 × 顕在
c = 上位構造 × 潜在
d = 上位構造 × 顕在
```

この2軸がどれくらい絡み合っているかを `entanglement` として測ります。

絡み合いが強い場合、たとえば、

```text
自分の本音だけを切り離して扱えない
家族、年齢、仕事、社会規範、外部事件と一体で動いている
```

という読みになります。

## 2つのモード

サイトの最初で、読み方のモードを選びます。

### 一般ヴァージョン

人生相談、物語分析、創作、キャリア、恋愛、家族、社会的テーマなどに使う汎用モードです。

| 成分 | 意味 |
|---|---|
| `a` | 内的核。まだ十分に現れていない本音、願い、才能、傷、恐れ、可能性 |
| `b` | 現実相。生活、身体、仕事、家庭、役割、習慣、制約、現在の状況 |
| `c` | 背後秩序。価値観、人生観、制度、因果、テーマ、深層パターン |
| `d` | 顕在作用。出会い、事件、支援、喪失、転機、外部からの働き |

### スピリチュアル・神秘主義・求道者ヴァージョン

霊的体験、信仰、悟り、神秘主義、宗教思想、求道プロセスを読むためのモードです。

| 成分 | 意味 |
|---|---|
| `a` | 魂的個我 |
| `b` | 顕現した個我 |
| `c` | 非顕現の神・真理 |
| `d` | 顕現した神性・恩寵 |

求道者版は、一般版の特殊ケースです。

```text
内的核      → 魂的個我
現実相      → 顕現した個我
背後秩序    → 非顕現の神・真理
顕在作用    → 顕現した神性・恩寵
```

## 使い方

### 1. モードを選ぶ

サイトで、一般ヴァージョンか求道者ヴァージョンを選びます。

### 2. AI変換プロンプトをコピーする

サイトのボタンから、選択中のモードに対応したAI変換プロンプトをコピーします。

### 3. 自分の物語をAIに渡す

ChatGPT、Gemini、Claudeなど、自分の使いたいAIにプロンプトと物語を貼ります。

AIは `config JSON` を返します。

### 4. config JSONをサイトに貼る

サイトがブラウザ内で測定します。  
外部APIには送信しません。

### 5. AI解釈専用JSONをコピーする

測定後、サイトは `ai_interpretation_v3` JSON を作ります。

これには、AIが深く読むための情報が入っています。

- `component_meanings`
- `life_question`
- `expected_reading_full`
- `gates_summary`
- `gate_resonance`
- `entanglement`
- `classical_controls`
- `gate_trace`
- `ablation`
- `order_sensitivity`
- `phase_sensitivity`

### 6. AIに結果を読ませる

サイトの「AI解釈プロンプト + AI解釈専用JSON」をコピーして、AIに貼ります。

AIは、単なる数値表ではなく、次のような構成で読みます。

```text
回路が見たあなたの物語
語りと構造のずれ
蝶番の出来事
静かな種と、洗い流されたもの
順序は運命だったか
通り方の質は結果を変えたか
二つの問いの絡み合い
この読みの限界
持ち帰る問い
```

## 回路が出す主な診断

### `probabilities`

最終状態の4成分確率です。

```text
どの成分が最終的に強く残ったか
```

を見ます。

### `observed_ranking_from_probabilities`

確率から見た実測順位です。

`expected_reading_full.ranking` と比べることで、語りと構造のずれを見ます。

### `gate_trace`

ゲートごとの変化です。

```text
どの時点で、どの成分が増えたか
どの時点で、どの成分が減ったか
```

を追えます。

### `ablation`

ゲートを1つ抜いた場合、最終結果がどれだけ変わるかを見ます。

これは、

```text
その出来事がなかったら、どれだけ違う物語になっていたか
```

を見る反実仮想です。

### `gate_resonance`

このツールの重要な診断です。

各ゲートについて、

```text
起きた瞬間の効果
最終結果への重み
```

を比較します。

| ラベル | 意味 |
|---|---|
| `QUIET_SEED` | その瞬間は小さいが、後から大きく効いた静かな種 |
| `WASHED_OUT` | その瞬間は大きいが、後の流れで洗い流されたもの |
| `AMPLIFIED` | 即時効果も大きく、最終的な重みもさらに大きいもの |
| `PROPORTIONATE` | 即時効果と最終的な重みが釣り合っているもの |
| `DORMANT_BUT_STRUCTURAL` | その瞬間はほぼ動かないが、干渉経路として構造的に効いたもの |
| `NEGLIGIBLE` | ほとんど影響しなかったもの |

### `order_sensitivity`

隣接する出来事の順番を入れ替えたとき、結末がどれだけ変わるかを見ます。

### `phase_sensitivity`

出来事の通り方を変えたとき、結末がどれだけ変わるかを見ます。

### `entanglement`

2つの軸の絡み合いを見ます。

純粋状態の concurrence は次で計算します。

```text
C = 2 | z_a z_d - z_b z_c |
```

| Concurrence | 判定 |
|---|---|
| `C < 0.1` | `SEPARABLE_LIKE` |
| `0.1 ≤ C < 0.5` | `WEAKLY_ENTANGLED` |
| `0.5 ≤ C < 0.9` | `STRONGLY_ENTANGLED` |
| `0.9 ≤ C` | `NEAR_MAXIMAL` |

### `classical_controls`

量子っぽい構造が本当に効いているかを確認するための対照です。

- `phase_dependence`: 全ゲートの位相を `phi=0` にしたとき、結果がどれだけ変わるか
- `interference_gap`: 位相・干渉を使わない古典マルコフ遷移と比べて、結果がどれだけ違うか

両方がLOWなら、そのconfigでは量子的構造の効きは弱く、古典的な因果要約に近い読みになります。  
どちらかがMEDIUM/HIGHなら、位相や干渉が結果に効いています。

## ゲートの読み方

6種類のゲートは、一般版では次のように読めます。

| ゲート | 一般版の読み | 求道者版の読み | 構造 |
|---|---|---|---|
| `G_ab` | 内的核 ↔ 現実相 | 魂的個我 ↔ 顕現した個我 | 顕現軸の局所回転 |
| `G_cd` | 背後秩序 ↔ 顕在作用 | 非顕現の神 ↔ 顕現した神性 | 顕現軸の局所回転 |
| `G_ac` | 内的核 ↔ 背後秩序 | 解脱・分霊 | 主体軸の局所回転 |
| `G_bd` | 現実相 ↔ 顕在作用 | 帰依・恩寵 | 主体軸の局所回転 |
| `G_bc` | 現実相 ↔ 背後秩序 | 求真・啓示 | 交換項 |
| `G_ad` | 内的核 ↔ 顕在作用 | 悟り・召命 | 対生成項 |

特に `G_bc` と `G_ad` は面白いゲートです。

```text
G_bc = 現実の出来事が、意味や構造の問いへ変わる
G_ad = 外部の出来事と内的核が同時に立ち上がる
```

## config JSONと測定結果は違う

このツールでは、AIの役割とサイトの役割を分けています。

```text
AIエンコーダ: 物語 → config JSON
ブラウザ計算: config JSON → 測定結果
AI解釈者: 測定結果 → 物語の言葉への翻訳
```

`config JSON` は測定前の設計図です。  
`probabilities` や `gate_resonance` や `entanglement` を含む結果JSONとは違います。

AI解釈には、必ず測定後の `result JSON` または `AI解釈専用JSON` を使ってください。

## エンコーダ再現性を見る

同じ物語を何度かAIにエンコードさせると、毎回まったく同じconfigになるとは限りません。

そこで、複数のconfigを比較するCLIがあります。

```bash
node tools/encoder_variance.mjs a.json b.json c.json
```

これにより、ゲート、強度、位相、順位、確率分布、concurrence のばらつきを確認できます。

これは、

```text
物語の構造を測っているのか
AIの気分を測っているのか
```

を確認するための重要なチェックです。
