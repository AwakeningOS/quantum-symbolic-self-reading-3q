# 量子物語回路 3Q

**あなたの物語を、3つの問いを持つ量子回路に通してみよう。**

人生、恋愛、仕事、創作、神話、偉人の物語、霊的体験。  
ふつうに読むと「なんとなくそういう話」で終わる物語を、8つの象徴へ分け、ブラウザ上の3量子ビット回路で測定します。

公開サイトはこちらです。  
**https://awakeningos.github.io/quantum-symbolic-self-reading-3q/**

このツールは、未来を当てる占いでも、霊的真実の証明でもありません。  
**あなたが書いた物語の中で、どの力が残り、どの出来事が流れを変え、過去と未来のどちらへ重心が向いているかを見るための実験的な物語リーダーです。**

## まず何をすればいい？

サイトを開いたら、基本はこの流れです。

1. **モードを選ぶ**  
   人生・恋愛・仕事・創作なら `general`、霊的体験・信仰・悟りの物語なら `seeker` を選びます。

2. **物語を書く**  
   悩み、出来事の順番、本音、外から来た変化、まだ言葉になっていない願いを書きます。

3. **AI変換プロンプト+記入内容をコピーする**  
   ボタンを押すと、AIに渡すためのプロンプトと本文がまとめてコピーされます。

4. **ChatGPT / Claude / Gemini / ローカルLLM などに貼る**  
   AIが `config JSON` を作ります。

5. **config JSON をサイトに貼って測定する**  
   ブラウザ内で量子回路が動き、8成分の確率、三軸の絡み合い、分岐点の出来事などを計算します。

6. **AI解釈プロンプト + AI解釈専用JSONをコピーする**  
   もう一度AIに貼ると、測定結果を日本語で解説できます。

## これは何をしているツール？

このツールは、物語を次のように扱います。

```text
物語を書く
↓
AIが物語を「8つの象徴」と「ゲート列」に変換する
↓
サイトが3量子ビット回路として計算する
↓
AIが測定結果を物語の言葉に戻す
```

ここでいう「量子回路」は、物理的な量子コンピューターを使うという意味ではありません。  
ブラウザ上で、複素振幅・位相・干渉・絡み合いを持つ小さな量子回路をシミュレーションしています。

なぜ量子回路にするのか。  
物語には、単純な足し算では読めない構造があるからです。

- ある出来事は、その瞬間は小さく見えても、あとで大きく効くことがある。
- 逆に、当時は大事件だったのに、後の流れで洗い流されることがある。
- 同じ出来事でも、受け入れて通るか、抵抗して通るかで結末が変わることがある。
- 自分の本音と、社会・家族・時代・神性のような大きな流れが切り離せないことがある。
- 表向きは未来へ進んでいるように見えても、構造的には過去に引かれていることがある。

このような「あとから効く」「打ち消し合う」「絡み合う」「過去と未来が綱引きする」構造を、量子回路の形で見えるようにしています。

## 3量子ビット化とは？

3Q版では、物語を3つの問いで読みます。

| 軸 | 問い | general | seeker |
|---|---|---|---|
| 主体軸 | 誰の物語として動いているか | 当事 ↔ 世界 | 個我 ↔ 超越 |
| 顕現軸 | それは形になっているか | 潜在 ↔ 顕在 | 非顕現 ↔ 顕現 |
| 時間軸 | どちらの時を向いているか | 過去 ↔ 未来 | 過去 ↔ 未来 |

この3つの問いを掛け合わせると、2×2×2で **8つの状態** になります。

```text
a0, a1, b0, b1, c0, c1, d0, d1
```

数字の `0` は過去側、`1` は未来側です。  
たとえば `a0` は「当事者の内側にある、過去から来たもの」、`a1` は「当事者の内側にある、未来へ向かうもの」です。

技術的には、ビット対応は次の通りです。

```text
index = 4*q1 + 2*q2 + q3
q1 = 主体軸
q2 = 顕現軸
q3 = 時間軸
```

でも、普通に使うときはこの式を覚える必要はありません。  
大事なのは、**3Q版では「本音か現実か」だけでなく、「それが過去に引かれているのか、未来へ向かっているのか」まで読める** という点です。

## ゲートとは？

ゲートは、物語の中で **ある力が別の力へ流れる出来事** です。

たとえば general なら、

```text
過去の傷 a0 が、未来への願い a1 に向き直る
```

これは `a0 → a1` のゲートです。  
時間軸の向きが変わるので、これは **転回系** です。

主なゲート族は次の通りです。

| ゲート族 | つなぐもの | 意味 |
|---|---|---|
| 表出系 | a ↔ b | 内側の願い・傷が、現実の行動や生活に出る |
| 発現系 | c ↔ d | 背後の構造が、現実の出来事として現れる |
| 共鳴系 | a ↔ c | 内側の核と、背後の秩序が響き合う |
| 遭遇系 | b ↔ d | 現実の自分が、外から来た出来事に出会う |
| 転回系 | *0 ↔ *1 | 過去向きの重みが未来へ向き直る、または逆に戻る |
| 役割系 | b ↔ c | 現実の役割と、社会・信念・真理の構造が結びつく |
| 呼応系 | a ↔ d | 内なる願いと、外から来た出来事が呼応する |

3Q版で特に大事なのは、**転回系** です。  
これは「昔の傷が未来の願いに変わる」「過去の生き方が新しい生き方に向き直る」「古い教えが未来の兆しになる」といった、時間の向きが変わる瞬間を表します。

## 位相とは？

位相は、出来事をどんな質で通ったかを表します。

| 位相 | ざっくりした意味 |
|---|---|
| 0 | 素直につながる、強め合う |
| ±π/2 | まだ統合されていない、直交している |
| π | 抵抗、反転、打ち消し |

たとえば、同じ「出会い」でも、素直に受け取った出会いと、怖くて拒んだ出会いでは、物語の行き先が変わります。  
この違いを回路では位相として扱います。

## 測定結果で見るポイント

測定後にたくさんの数値が出ますが、最初に見るのはこのあたりです。

### 1. どの成分が強く残ったか

`probabilities` は、回路を通したあとに8成分のどこへ重心が残ったかを示します。

`a1 志向` が強ければ「まだ形にならない未来への願い」が残っている。  
`c0 慣性` が強ければ「世界や規範の古い構造」がまだ強く残っている。  
`b1 新生` が強ければ「新しい生き方」が立ち上がっている、という読みになります。

### 2. 予想と実測がズレたか

AIが作る config には `expected_reading` があります。  
これは「こうなるだろう」という仮説です。

でも、回路で測ると違う順位になることがあります。  
これは失敗とは限りません。

むしろ、

```text
あなたが語っている結末
vs
あなたが語った出来事の列が構造的に導いた結末
```

のズレとして読むと面白い部分です。

### 3. 分岐点の出来事

`gate_resonance` は、どの出来事を取り除くと結末が大きく変わるかを見ます。

つまり、**物語の分岐点になった出来事** を探します。  
目立つ大事件とは限りません。小さな一言、地味な出会い、あとから効いてくる違和感が、実は最大の分岐点になることがあります。

### 4. 静かな種と洗い流されたもの

`QUIET_SEED` は、その瞬間は小さく見えたのに、後から大きく効いた出来事です。  
`WASHED_OUT` は、その瞬間は大きかったのに、後の流れで効果が薄れた出来事です。

これは普通に物語を読むだけでは見えにくい、回路ならではの読みどころです。

### 5. 時間の向き

`bloch_z.time` と `axis_populations` は、物語が過去寄りか未来寄りかを見ます。

- 正なら、過去寄り
- 負なら、未来寄り

ただし、少し傾いているだけなら「やや過去寄り」「やや未来寄り」と読みます。  
このツールでは、強い断定を避けるため、数値の大きさに応じて表現を調整します。

## 三つの問いの結び方

3Q版では、三つの軸がどのように絡み合っているかも見ます。

| ラベル | 意味 |
|---|---|
| GHZ_KNOT | 三つの問いが一つの結び目になっている。どれか一つだけ切り離して解けない |
| W_WEAVE | 対ごとの綾で編まれている。二つずつなら扱える |
| HYBRID | 結び目と綾の混合 |
| SEPARABLE_LIKE | 三つの問いはほぼ独立に扱える |

たとえば `GHZ_KNOT` なら、

```text
自分は誰か
それは現実に出ているか
それは過去と未来のどちらを向いているか
```

この三つが一つの結び目になっています。  
「仕事だけ変えればいい」「恋愛だけ考えればいい」というように、一部だけ切り離して扱いにくい構造です。

`W_WEAVE` なら、三つが完全な一つの結び目ではなく、二つずつ絡んでいます。  
たとえば「現実の行動」と「時間の向き」は絡んでいるが、「当事者と世界」の絡みは弱い、という読みができます。

## Gate flow / 流れの健全性

3Q版では、8状態あるため、まだ流れが届いていない成分が多くなります。  
そのため、AIが意味だけでゲートを作ると、回路上は何も起きないゲートが生まれることがあります。

この事故を防ぐため、サイトは `gate_flow` を監査します。

| 判定 | 意味 |
|---|---|
| NORMAL | source に流れが届いており、通常通り働いた |
| NO_OP | source と target の両方が空で、回路上ほぼ何も起きなかった |
| SOURCE_EMPTY | source が空で、意図と逆向きに target から source へ流れた |

全体の健康状態は次のように出ます。

| 判定 | 意味 |
|---|---|
| HEALTHY | 問題ゲート0本 |
| DEGRADED | 問題ゲート1〜2本 |
| COMPROMISED | 問題ゲート3本以上 |

`DEGRADED` や `COMPROMISED` が出た場合、その測定は使えないとは限りません。  
ただし、いくつかの出来事が測定に反映されていない、または逆向きに働いた可能性があります。  
その場合は、AIに再エンコードさせてからもう一度測定することを推奨します。

## 古典対照

このツールは、量子っぽい言葉を使うだけでなく、古典的な確率遷移でも同じ結果にならないかを確認します。

- `phase_dependence`: 位相を全部0にしたら、結果がどれだけ変わるか
- `interference_gap`: 干渉なしの古典マルコフ遷移と、結果がどれだけ違うか

両方が `LOW` なら、その config は量子的な位相・干渉を使わなくても、ほぼ古典的に説明できます。  
逆に `MEDIUM` や `HIGH` が出るなら、位相や干渉が結果を動かしている可能性があります。
